# AI Platform Transformation Proposal
## From Gemini Chatbot → AI Operating System

---

## Current State Summary

The app is a **fully functional Gemini chatbot** with:
- 8 AI models (Gemini + Gemma), streaming chat, threads, system prompts
- Image/voice upload, message editing, regeneration/branching
- Stripe subscriptions (Free/Pro/Team), admin dashboard
- Full-text search, export, sharing, context compression
- EN/RU i18n, 8 visual themes, session statistics
- Supabase auth (JWT), role-based access, rate limiting

**Stack**: React 19 + Vite 8 / Express 5 + Prisma + PostgreSQL / Supabase

## Phase 1: Foundation (Weeks 1-4)
**Goal**: Keep chatbot running, add ComfyUI image generation, build execution infrastructure

### 1.1 Backend Refactor — Split monolith into services
- `server.js` (1185 lines) → split into route modules:
  - `routes/auth.js`, `routes/threads.js`, `routes/chat.js`, `routes/subscription.js`, `routes/admin.js`
- Add **Task Queue** system (BullMQ + Redis) for async job processing
- Add **Execution State** tracking in DB (new `Execution` model)

### 1.2 ComfyUI Integration
- Add ComfyUI client service (`services/comfyui.js`)
- New API endpoint: `POST /api/generate/image` — sends workflow to ComfyUI
- New API endpoint: `GET /api/executions/:id` — poll execution status
- SSE streaming for generation progress updates
- Store generated images in Supabase Storage

### 1.3 Credits System
- New `Credit` model in DB (balance, transactions log)
- Free: 100 credits/day, Pro: 1000/day, Team: unlimited
- Each AI call costs credits based on model + task type
- Credit balance visible in StatsDashboard

### 1.4 Database Additions
```prisma
model Execution {
  id          String   @id @default(uuid())
  userId      String
  type        String   // "chat" | "image" | "video" | "workflow"
  status      String   // "pending" | "running" | "completed" | "failed"
  provider    String   // "gemini" | "comfyui" | "runpod"
  input       Json
  output      Json?
  creditsUsed Int
  startedAt   DateTime
  completedAt DateTime?
  createdAt   DateTime @default(now())
}

model Credit {
  id        String @id @default(uuid())
  userId    String @unique
  balance   Int    @default(100)
  updatedAt DateTime @updatedAt
}

model CreditTransaction {
  id        String   @id @default(uuid())
  userId    String
  amount    Int      // negative = spend, positive = add
  reason    String   // "daily_grant" | "chat" | "image_gen" | "purchase"
  executionId String?
  createdAt DateTime @default(now())
}
```

### 1.5 Frontend — Generation Panel
- New component: `GenerationPanel.tsx` — sidebar or modal for image generation
- Image prompt input, model/style selector, resolution picker
- Generation history grid ( thumbnails of past generations )
- Download / regenerate / use-in-chat buttons

**Deliverables**: Chatbot still works. Users can now generate images via ComfyUI. Credits system active.

## Phase 2: Workflow System (Weeks 5-8)
**Goal**: Visual workflow editor, saved pipelines, multi-step execution

### 2.1 Workflow Engine
- New `Workflow` model (name, description, graph JSON, isPublic, authorId)
- Workflow graph format: nodes + edges (React Flow compatible)
- Node types: `input`, `text_gen`, `image_gen`, `video_gen`, `audio_gen`, `transform`, `output`
- Each node maps to a backend execution endpoint

### 2.2 Workflow Executor
- Backend service that walks the graph, executes each node, passes outputs to next nodes
- Parallel execution where graph allows
- Checkpoint/resume capability
- Error handling per node (retry, skip, fallback)

### 2.3 Frontend — Workflow Editor
- React Flow integration for visual node editor
- Drag-and-drop node palette
- Node configuration panels
- Real-time execution status overlay on nodes
- Save/load workflow templates

### 2.4 Workflow API
- `POST /api/workflows` — create
- `GET /api/workflows` — list user's workflows
- `PUT /api/workflows/:id` — update graph
- `POST /api/workflows/:id/execute` — run workflow
- `GET /api/workflows/:id/executions` — execution history

**Deliverables**: Users can build visual AI pipelines, save them, and execute multi-step workflows.

## Phase 3: Agent Layer (Weeks 9-12)
**Goal**: Autonomous AI agents that can plan, execute, and coordinate

### 3.1 Agent Framework
- Agent base class with: `plan()`, `execute()`, `observe()`, `reflect()`
- Agent types: `PlannerAgent`, `GeneratorAgent`, `EditorAgent`, `QAAgent`, `PublisherAgent`
- Agent communication via message queue
- Shared memory/context between agents

### 3.2 Agent Orchestrator
- Receives high-level intent from user
- Decomposes into subtasks via PlannerAgent
- Dispatches subtasks to specialized agents
- Monitors progress, handles failures
- Aggregates results

### 3.3 Agent UI
- Live agent status dashboard (who's doing what)
- Agent conversation log (inter-agent communication)
- Manual intervention points (pause, redirect, override)
- Agent performance metrics

### 3.4 Example: "AI Video Factory" Pipeline
```
User: "Create a 30-second sci-fi trailer"
  → PlannerAgent: decomposes into scenes
  → ScriptAgent: writes dialogue/narration
  → StoryboardAgent: creates shot list
  → ImageAgent: generates key frames (ComfyUI)
  → VideoAgent: animates frames (ComfyUI video nodes)
  → VoiceAgent: generates narration (TTS)
  → MusicAgent: creates background score
  → EditorAgent: assembles final video
  → QAAgent: reviews quality
  → PublisherAgent: exports final output
```

**Deliverables**: Users can describe a complex task in natural language, and a team of agents produces the result.

## Phase 4: Marketplace (Weeks 13-16)
**Goal**: Community marketplace for workflows, agents, and templates

### 4.1 Marketplace Backend
- `MarketplaceItem` model (name, description, type, price, authorId, downloads, rating)
- Types: `workflow`, `agent_template`, `style_preset`, `lora_pack`, `automation_template`
- Publishing workflow: draft → review → published
- Revenue split: 80% creator / 20% platform

### 4.2 Marketplace Frontend
- Brows/search/filter marketplace
- Preview workflow graphs before installing
- One-click install to user's workspace
- Rating and review system
- Creator profiles and dashboards

### 4.3 Monetization
- Credits purchasable via Stripe (packs: 500/$5, 2000/$15, 10000/$40)
- Marketplace takes 20% commission on paid items
- Free community items available
- Enterprise: custom pricing for private marketplace instances

**Deliverables**: Ecosystem where creators publish AI workflows, users install them, platform earns revenue.

## Risk Mitigation

| Risk | Mitigation |
|------|------------|
| ComfyUI not available locally | Use ComfyUI cloud (RunPod/Lightning.ai) or mock for dev |
| Redis dependency | Optional: use in-memory queue for MVP, Redis for production |
| Scope creep | Phase-gated: don't start Phase N+1 until Phase N is stable |
| Current chatbot breaks | All changes are additive — existing routes stay functional |
| Cost of cloud GPU | Start with pay-per-use (RunPod), optimize caching later |
